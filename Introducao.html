Introdução - NoSQL

Os bancos de dados NoSQL, também chamados de não estruturados/ não relacionais, são bancos que não possuem dados estruturados, ou seja, não trabalham com dados que possuem tabelas e linhas como identação. Os bancos NoSQL foram desenvolvidos para a manipulação de grandes quantidades de dados, estando muito em contexto e ligado ao Big Data, onde grandes massas de dados são gerados constantemente. Logo, os bancos não relacionais são bancos distribuídos que foram projetados para atender os requerimentos exigidos por situações como o Big Data, oferecendo uma arquitetura onde a linguagem SQL não é cabível com suas condições.

Pela especulação de Big Data ser o novo futuro no ramo de TI, os bancos NoSQL estarão em grande massa no mercado, onde alguns ousam dizer até mesmo que pode substituir os bancos tradicionais relacionais. Portanto, estes bancos oferecem 4 categorias principais de bancos, sendo eles:

  1. Graph Databases

  2. Document Databases

  3. Key-Values Stores

  4. Column Family Stores

  • Graph Databases

    • Esta categoria é geralmente aderente a cenários de rede social online, onde os nós representam e os laços representam as interconexões entre eles. Tendo sido usada para lidar com problemas relacionados a sistemas de recomendação e listas de contreole de acesso, fazendo uso de sua capacidade de lidar com dados altamente interligados.

  • Document Databases

    • Estes permitem o armazenamento de uma grande massa de documentos, através de associação de dados coletados. Um exemplo deles, é o MongoDb.

  • Key-Value Store

    • Neste caso, os dados são armazenados no formato chave-valor, onde os valores/dados são identificados por suas chaves. É possível armazenar inúmeros registros de forma eficiente e o processo de escrita é bem rápido, além de poder pesquisa-los através de chaves associadas.

  • Column Family Store

    • Onde os dados são orientados a colunase organizados em grupos dessas colunas. Tanto o armazenamento quanto as pesquisas de dados são baseados em chaves.

Outras características dos dados NoSQL são: a representação de dados sem schema, tempo de desenvolvimento, velocidade e escalabilidade

Para uma lista completa de bancos de dados NoSQL e em quais categorias eles se encontram, visite o site NoSQL-database.org, mas também seguem uma explicação geral sobre os mais usados:

  • MongoDB

    • Sendo um banco orientado a documento, onde substitui o conceito de linha por documentos em termos de armazenamento dos dados, sendo também open-source e possuindo características como: indexação (onde o Mongo suporta índices secundários, permitindo a construção de Querys velozes), Agregação (permitindo a construção de agregações complexas e dados, otimizando o desempenho), Tipos de dados especiais (Suporta coleções time-to-live para dados que expiram em um determinado tempo, como sessões e etc.), além de conceitos semelhantes aos bancos SQL:
